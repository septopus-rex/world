# Septopus Protocol

Welcome to the **Septopus Protocol** documentation index. The Septopus Engine is a 3D execution platform built upon **Adjuncts** (附属物). The **String Particle Protocol (SPP)** is a semantic space organization protocol that acts as a highly compressed "announcer," deployed via specialized meta-adjuncts to resolve complex spatial structures for the engine.

This directory contains the core specifications for the various components and interactions within the Septopus engine.

To accommodate our global community, the protocol documentation is available in both English and Chinese.

## 📚 Protocol Specifications

Please navigate to your preferred language directory above to access the detailed documentation for the following protocol components:

*   **Adjunct Protocol**: Defines "Adjuncts" (附属物) as the primary mechanism for extending the engine's capabilities and rendering dynamic, interactive 3D objects.
*   **Animation Protocol**: Covers the animation system (`/animation`) used to describe and control dynamic behaviors.
*   **Block Protocol**: Explains the core Septopus "Block" structure, the foundational unit of space in the engine.
*   **Framework Protocol**: Outlines the high-level architecture and the Septopus Sandbox environment.
*   **Player Protocol**: Details the definitions and interactions related to user representations (avatars) and inputs within the world.
*   **Trigger Protocol**: Describes the Trigger system used for spatial logic, event detection, and interactivity.
*   **World Protocol**: Defines the top-level configuration and initialization parameters for a Septopus World instance.
