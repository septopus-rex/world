# Septopus 世界总览(Overview)

> **从这里开始。** 本文用一页讲清 Septopus 世界**由什么构成**、各部分**如何组合**、
> 以及**为什么这样设计**。读完再进各分册看细节。规范级细节散见 [world](world.md)/
> [block](block.md)/[adjunct-types](adjunct-types.md)/[trigger](trigger.md)/
> [determinism](determinism.md);实现细节在 [`docs/`](../../docs/)。

## 1. 一句话

Septopus 是一个**跨引擎、纯数据的 3D 世界协议**:世界的一切——地形、物体、
交互、玩法、时间、天气——都是**带确定性语义的纯数据**。引擎只做一件事:把这份
数据解成一个可走、可玩的三维世界。**同一份数据,任何合规引擎解出同一个世界。**

## 2. 空间构成:世界 ← 地块 ← 附属物

三层空间容器,从大到小:

```
宇宙(6 面 × 4×4 = 96 个世界)
  └─ 世界 World       4096 × 4096 地块的有界网格,有"领主"(Lord)主权
       └─ 地块 Block   16m × 16m 的原子空间容器,可拥有、可交易(数字地产)
            └─ 附属物 Adjunct   放在地块上的一切内容(墙/物品/NPC/触发器/光…)
```

- **世界(World)**:最高边界框,强制全局物理与大气规则;每个世界有一个领主
  (持 NFT/密钥),可调地形基色、默认海拔、允许的模式等。详见 [world.md](world.md)。
- **地块(Block)**:世界的**基本单元与真正的资产**——16×16 米,单一所有者,
  可交易、可遗弃重认领。它是空间锚点:附属物的坐标都相对地块原点(西南角),
  地块动则其上一切随动。地块内容是一个**五元组** `[海拔, 状态, 附属物, 动画, 可玩标记]`。
  详见 [block.md](block.md)。
- **附属物(Adjunct)**:地块上的内容原子——18 种内置类型(几何/光/模型/水/物品/
  NPC/触发器/生成器/媒体…),每种是一行紧凑的 `[typeId, 数据槽…]` raw。附属物与引擎
  完全解耦:可从本地、IPFS 或链上加载。详见 [adjunct-types.md](adjunct-types.md)。

**为什么是"地块"这个粒度?** 16 米方格既是**产权单位**(可单独拥有/交易/发布),
又是**流式加载单位**(引擎只加载玩家周围的地块,4096×4096 的世界不撑爆内存),
还是**内容寻址单位**(一块 = 一份可 diff、可上链、可分享的数据)。产权、性能、
存储三件事共用同一个边界,是整个架构的地基。

## 3. 内容构成:一切都是数据词汇

Septopus 的核心信条是**「数据即逻辑」**——创作者永不写代码,只填数据:

| 你想要 | 你填的数据 |
|---|---|
| 一堵墙 / 一盏灯 / 一个模型 | 一行 adjunct raw(类型 + 尺寸/位置/资源) |
| "踩到就开门" / "有钥匙才传送" | 触发器(事件 + JSONLogic 条件 + 动作),见 [trigger.md](trigger.md) |
| 一个会走会打的怪 | NPC 行为文档(数据状态机) |
| 一段对话 / 一个任务 | 对话树 + flags 配方(零新原语) |
| 一片村庄 / 一栋楼 | 生成式内容(模板 + seed,确定性展开) |

这样做的回报是**跨引擎**:数据不含任何引擎特定的代码或几何,所以换一个渲染引擎
(UE/Babylon/自研)只需重写"解释器",已存世界内容一字不改。参考实现(本仓 TS 引擎)
把渲染严格隔离在一层里,正是为守住这条线。

## 4. 环境构成:时间与天气是**推导**出来的,不是存出来的

世界不存"现在几点、什么天气"这种状态,而是**从确定性源纯函数推导**:

```
链高度 + 链 hash + 出块间隔  ──纯函数──▶  年/月/日/时 + 当前天气
```

- **时间**:世界时钟由链高度换算(链历法),昼夜、晨昏由此派生;链历法跳变时
  引擎平滑追赶,不弹跳。
- **天气**:从链 hash 的切片确定性派生(晴/雨/雷暴 + 等级),同高度同 hash → 同天气。

**为什么不直接存?** 因为要**跨引擎一致 + 无需信任**:任何引擎、任何时刻,拿同样的
(高度, hash) 都算出同一个世界时刻与天气,不需要一个中心服务器来"广播现在的天气"。
这与物品实例、生成式内容用 seed 确定性推导是同一个哲学。规范见
[world.md §3.1](world.md) 与 [determinism.md](determinism.md)。

## 5. 玩法构成:五种模式 + 数据驱动的交互回路

- **模式**:Normal(自由行走)/ Edit(放置编辑)/ Game(区域门控的玩法)/
  Ghost(穿墙飞行)/ Observe(环绕观察)。Game 不是自由开关——由地块的"可玩标记"
  门控,进入是仪式化的(走到触发器 → 确认)。
- **交互回路**:玩家的点击/碰撞 → 触发器/NPC 评估条件 → 执行**动作**(actuator:
  施伤、给物品、传送、生成、播音…)→ 改世界状态(flags/背包/生命)→ 又成为下一个
  条件的输入。整个回路是数据 + 通用系统,零专用玩法代码——一个完整的仙剑风 RPG
  就是这么用纯数据关卡搭出来的。

## 6. 存储构成:本地优先,链可选

- **本地优先**:编辑存本地草稿(IndexedDB),发布进内容寻址存储(CID)。零钱包、
  零网络即可创作与游玩。
- **链可选**:链作为**可选发布插件**存在(`IChainPublisher` 注入),不装则纯本地
  运行,引擎核心零链依赖。地块的产权/发布是链能力,内容的创作/游玩不需要链。

这就是为什么本引擎能"独立"——链是它的一个可选出口,不是它的运行前提。

## 7. 为什么这样设计(一页回顾)

| 设计选择 | 为了什么 |
|---|---|
| 世界←地块←附属物 三层容器 | 产权、流式加载、内容寻址共用一个边界(16m 地块) |
| 一切都是数据词汇(数据即逻辑) | 跨引擎(换渲染器不改内容)+ 与人写内容同一安检链 + AI 可直接生成 |
| 时间/天气/物品/生成 用 seed/链源确定性推导 | 跨引擎一致 + 无需中心服务器 + iNFT 性质 |
| 五模式 + 区域门控 Game | 空间访问与玩法节奏有序,不靠自由开关 |
| 本地优先、链可选 | 独立运行是常态,上链是可选叠加 |

一句话:**内容=协议数据,几何=确定性配方,时间/天气=纯函数,渲染=可插拔末端**——
四层各自独立演化,这就是 Septopus 3D 引擎"长这样"的原因。
