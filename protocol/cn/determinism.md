# Septopus 确定性规范(Determinism & Conformance)

> **规范级(Normative)**。「跨端、纯数据 3D 世界」成立的前提是**同一份数据在任何
> 引擎里解出同一个世界**。本文集中钉死所有确定性推导点(钉点),并给出一致性
> (conformance)验收清单。协议禁止在这些推导中使用墙钟时间、`Math.random`、
> 浮点平台差异敏感的路径。

## 1. 基准 PRNG:mulberry32

所有 seed 驱动的推导统一使用 mulberry32。**参考实现即规范**(逐位一致):

```js
function mulberry32(seed) {
    let a = seed >>> 0;
    return () => {
        a = (a + 0x6d2b79f5) >>> 0;
        let t = a;
        t = Math.imul(t ^ (t >>> 15), t | 1);
        t ^= t + Math.imul(t ^ (t >>> 7), t | 61);
        return ((t ^ (t >>> 14)) >>> 0) / 4294967296;
    };
}
```

- seed 一律先 `>>> 0` 规格化为 uint32。
- 序列消耗次数属于规范:各钉点文档标明"恰好消耗 N 次 rng"处,实现不得多取或少取
  (否则后续序列整体错位)。

## 3. 一致性验收(conformance checklist)

一个自称实现 Septopus 协议的引擎,应通过以下验收(与参考实现比对):

1. **解码一致**:同一地块 raw(全 18 型样例)→ 实体集合(类型/位置/尺寸/属性)一致;
   可省尾槽缺失不报错、取表定缺省值。
2. **PRNG 一致**:`mulberry32(42)` 前 8 个输出与参考实现逐位一致。
3. **物品一致**:`(template=1, seed=777)` 等样例推导出的稀有度/属性与参考一致(P1)。
4. **展开一致**:同 `(template, seed, params)` 的 motif 展开行组逐字节一致(P4);
   同 cells 的 SPP 展开实体组一致(P5)。
5. **时间/天气一致**:同 `(height, hash)` 推导出同历法与天气(P2)。
6. **行走语义**:AABB 碰撞 + step-over(≤0.5m)+ b4 三形状(含坡面连续行走)
   在样例场景中产生可比路径(观感等价;逐帧位一致不作要求——见 §4)。
7. **旋转观感**:旋转契约样例(各轴 90°/45°)渲染朝向与参考一致
  (见 [world.md](world.md) §坐标与旋转契约)。

## 4. 明确不要求逐位一致的部分

- **渲染像素**、光照、阴影——观感等价即可。
- **玩家运动积分的逐帧浮点值**——不同引擎的物理步进可有平台差异;规范要求的是
  语义(能走上的坡必须能走上、挡住的墙必须挡住),P7 的会话重放验证发生在
  **同一实现内**。
- 网络/资源加载时序(装载是异步的;世界状态收敛后应一致)。
