# 系统效率分析

基于三层架构（IPFS 全状态定义 + 链上塌陷状态 + 引擎运行时）的全面效率估算。

---

## 一、存储分层架构

弦粒子的数据按"是否确定"分层存储：

```
┌─────────────────────────────────────────────┐
│  Layer 1: 全状态定义（IPFS）                   │
│  "这个空间可以是什么样"                         │
│  包含所有 cell、候选项、模型引用、材质、bounds  │
│  存储成本：免费 / 极低                          │
├─────────────────────────────────────────────┤
│  Layer 2: 塌陷状态 + 放置信息（链上）            │
│  "这个空间被确定成了什么样，放在哪里"             │
│  IPFS CID + origin + layerId + 塌陷选择索引   │
│  存储成本：44B header + 4 bytes/cell           │
├─────────────────────────────────────────────┤
│  Layer 3: 运行时（引擎）                        │
│  读取 Layer1 + Layer2 → 展开 → 渲染             │
│  动态效果（破坏/特效/音效）→ 纯本地处理          │
│  存储成本：无（内存中）                          │
└─────────────────────────────────────────────┘
```

---

## 二、存储成本

### Layer 1：IPFS

| 组件 | 典型大小 | 说明 |
|------|---------|------|
| ParticleDefinition JSON | 2-50 KB | cells + faceOptions + metadata |
| 单个 Option 模型 (glb) | 10-500 KB | 简单构件 10KB，精细模型 500KB |
| 一个主题的全部模型 | 1-10 MB | 约 20-50 个 Option |
| 纹理贴图 | 50 KB - 2 MB | 按分辨率 |
| 音效资源 | 10-100 KB | 短音效 |
| **一套完整模板总计** | **2-15 MB** | |

```
IPFS 存储成本：
  Pinata 免费额度：1 GB        → 可存 60-500 套模板
  付费：$0.15/GB/月            → 1000 套 ≈ $1.5-2.25/月
  去中心化 pin (Filecoin)：    近乎免费
```

### Layer 2：链上

Header 固定 44 bytes + 每 cell 4 bytes：

| 场景规模 | Raw 大小 | RLE 压缩后 | L2 成本 (Base) | L1 成本 (ETH) |
|---------|---------|-----------|---------------|--------------|
| 小隔间 (6 cell) | 68 B | 68 B | < $0.01 | $1.50 |
| 标准房间 (18 cell) | 116 B | ~70 B | < $0.01 | $2.50 |
| 一层地牢 (100 cell) | 444 B | ~150 B | $0.01 | $7.50 |
| 大型地牢 (500 cell) | 2 KB | ~500 B | $0.02 | $20 |
| 城镇 (2000 cell) | 8 KB | ~2 KB | $0.05 | $80 |
| 巨型世界 (10000 cell) | 40 KB | ~10 KB | $0.20 | $400 |

> **在 L2（Base/Arbitrum）上，即使大型世界也只需几美分。ETH L1 仅适合小场景。**

### 新旧架构对比

| 指标 | 全链上 (旧 11B/cell) | IPFS+链上 (新 4B/cell) | 节省 |
|------|---------------------|----------------------|------|
| 100 cells raw | 1,104 B | 444 B | 60% |
| 100 cells RLE | ~300 B | ~150 B | 50% |
| 1000 cells RLE | ~2.5 KB | ~1 KB | 60% |
| Header 开销 | 4 B | 44 B | 多 40B（CID + 放置） |

Header 多出 40 bytes，但在 >10 个 cell 时就能由 cell 体积缩小完全弥补。

## 三、带宽成本

```
用户进入一个 Block 时需要加载的数据：

1. 链上读取：~100-500 bytes（塌陷状态）
   → RPC 调用，延迟 50-200ms，免费

2. IPFS 加载：~2-15 MB（首次加载全状态定义 + 模型）
   → 可缓存到 IndexedDB，后续无需重复下载
   → IPFS Gateway 延迟 100-500ms

3. 后续进入（相同模板）：仅需链上读取
   → 延迟 <200ms
```

| 场景 | 传统游戏 | 弦粒子系统 |
|------|---------|-----------|
| 首次加载地图 | 50-200 MB | 2-15 MB |
| 复用同模板 | 50-200 MB | < 1 KB |
| 热更新 | 重新下载整个包 | 仅链上塌陷状态变化 |

## 四、运行时效率

### 塌陷 + 展开

| 步骤 | 复杂度 | 100 cell | 1000 cell |
|------|--------|---------|----------|
| 解析链上二进制 | O(N) | < 0.1ms | < 0.5ms |
| 加载 IPFS 定义 | O(1) 缓存命中 | < 1ms | < 1ms |
| 塌陷选择映射 | O(N × 6) | < 0.5ms | < 3ms |
| 展开为 Adjunct | O(N × 6 × A) | ~5ms | ~30ms |
| 构建渲染对象 | O(M) | ~10ms | ~50ms |
| **总计** | | **~17ms** | **~85ms** |

> A = 每个构型平均包含的 Adjunct 数量（1-5 个），M = 最终渲染对象总数。

- 100 cell ≈ 17ms → 一帧内完成，无感知
- 1000 cell ≈ 85ms → 可分帧加载，5 帧内完成

### 渲染性能

| 场景规模 | Mesh 数量 | Draw Calls（合批后） | 目标 FPS |
|---------|----------|-------------------|---------|
| 小房间 (18 cell) | ~100 | 10-20 | 60 |
| 一层地牢 (100 cell) | ~400 | 30-60 | 60 |
| 大型地牢 (500 cell) | ~2000 | 100-200 | 30-60 |
| 城镇 (2000 cell) | ~8000 | 300-500 | 30 |

**优化手段：**

- **LOD**：远处用低 level 渲染，近处细分
- **视锥裁剪**：只渲染摄像机可见的 cell
- **实例化绘制**：相同构型合并为 instanced draw call
- **遮挡剔除**：被墙壁挡住的 cell 完全跳过

### 内存占用

| 组件 | 100 cell | 1000 cell |
|------|---------|----------|
| 塌陷状态（二进制） | 444 B | 4 KB |
| IPFS 定义（解析后） | ~50 KB | ~200 KB |
| 展开后 Adjunct 数据 | ~200 KB | ~2 MB |
| 渲染 Mesh 数据 | ~2 MB | ~20 MB |
| 纹理贴图 | ~10 MB | ~10 MB（共享） |
| **总计** | **~12 MB** | **~32 MB** |

> 现代设备 2-4 GB 可用内存，万级 cell 也在 ~200 MB 内。

## 五、AI 生成效率

| 任务 | 输入大小 | 输出大小 | LLM 耗时 |
|------|---------|---------|---------|
| 生成全状态定义 (100 cell) | 提示词 ~500字 | JSON ~10KB | 3-8s |
| 生成塌陷选择 (100 cell) | 定义 + 提示 | ~400B | 1-3s |
| 修改局部 (10 cell) | 局部上下文 | ~40B | <1s |

```
AI 的工作量对比：
  传统方式：生成 .obj/.fbx 模型   → 复杂，容易出错
  弦粒子：生成 JSON + 塌陷索引    → 简单离散选择，错误率极低
```

---

## 六、关键指标总结

| 指标 | 数值 | 意义 |
|------|------|------|
| 链上每 cell | **4 bytes** | 业内最小的链上 3D 空间表示 |
| L2 存储 100 cell | **< $0.01** | 忽略不计的链上成本 |
| 首次加载 | **2-15 MB** | 秒级加载，低于传统游戏 100x |
| 复用加载 | **< 1 KB** | 同模板不同塌陷，几乎零成本 |
| 展开耗时 100 cell | **~17 ms** | 单帧完成，无感知 |
| AI 生成一个房间 | **1-3 秒** | 快于人类操作 100x |
| 内存占用 | **12-32 MB** | 手机也能跑 |
