## 一、协议目标

SPP-Core 是一个：

> 面向 AI 的三维空间语义协议
> 用于描述“可塌陷”的空间结构

它不直接描述：

* 几何模型
* 材质
* 网格
* 引擎数据

而只描述：

```
空间节点
+
每个方向的可选空间关系
```

供：

* AI 布局
* 优化
* 塌陷生成具体3D结构

---

## 核心价值主张

### 1）维度无关

弦粒子描述的是空间语义（"这里有一扇门连接 A 和 B"），而不是几何体。因此同一份数据可以被不同引擎渲染为完全不同的维度：

```
同一份弦粒子数据
  ├── 3D 引擎    → 第一人称地牢探索
  ├── 2D 引擎    → 像素风 Roguelike
  ├── 2.5D 引擎  → 等距视角 RPG
  ├── ASCII 引擎 → 纯文本冒险
  └── VR/AR 引擎 → 沉浸式体验
```

同一个世界，不同设备、不同画面风格的玩家可以共存交互。

### 2）跨代适配

协议分离了语义、资产和渲染三层——只有资产和渲染会过时，语义永远有效：

```
传统游戏移植：
  逆向工程 → 重写引擎 → 重做资产 → 测试
  耗时数月到数年

弦粒子适配：
  链上数据 → 不变
  空间逻辑定义 → 不变（存在链上或持久化存储）
  新建 Option 资产映射 → AI 可自动生成
  新引擎读协议渲染 → 即刻适配
```

今天创建的弦粒子游戏，未来只需要一个**Option 映射表**（旧 ID → 新资产），就能在任何新设备上运行。

### 3）一次创作，永久可用

```
语义层（协议）      "这里有楼梯连接上下"     → 不会过时（100% On-Chain）
资产层（网络）      stair.glb 模型文件       → 可替换（CDN/IPFS）
渲染层（引擎）      WebGL / Vulkan / 未来    → 可替换
```

弦粒子是**空间领域的 HTML**——内容与表现分离，跨平台、跨时代。

## 二、弦粒子的本质定义

弦粒子（String Particle）是：

> 一个三维空间节点
> 每个方向包含一组“空间可选项”

这些可选项可以是：

* 墙
* 门
* 通道
* 楼梯
* 电梯
* 窗
* 管道
* 洞穴开口
* 能量门
* 任意风格结构

---

## 三、空间如何塌陷

在弦粒子系统中：

每个方向不是一个固定结构，而是：

```
一个候选集合
```

例如：

某个面的可选项：

```
[墙体A, 墙体B, 门, 窗]
```

系统在生成或优化时：

```
从集合中选择一个
```

这一步就是：

> 空间塌陷（Collapse）

---

## 四、最终极简数据结构

### ParticleCell

```
ParticleCell
├─ position: integer3
├─ level: integer(0–3)
├─ faceStates: bitmask6
└─ faceOptions: id[6][]
```

---

## 五、字段详细语义

### 1）position

```
[x, y, z]
```

三维整数坐标，用于粒子堆叠。

---

### 2）level

```
0–3
```

表示空间粒度层级：

| level | 空间尺度 |
| ----- | ---- |
| 0     | 家具级  |
| 1     | 房间级  |
| 2     | 建筑级  |
| 3     | 城市级  |

用于：

* 多尺度布局
* LOD
* AI规划

---

### 3）faceStates（6位）

表示该方向是否参与空间关系：

```
1 = 存在空间关系
0 = 完全封闭
```

方向定义：

```
0: +X
1: -X
2: +Y
3: -Y
4: +Z
5: -Z
```

---

### 4）faceOptions（核心字段）

结构：

```
faceOptions: id[6][]
```

含义：

```
6个方向
每个方向一个 id 列表
```

每个 id 表示：

```
一种空间可选结构
```

例如：

```
faceOptions[+Z] = [12, 45, 98]
```

表示：

这个面的三种可选塌陷结果：

* 12 → 楼梯
* 45 → 电梯
* 98 → 垂直管道

系统最终：

```
选择其中一个
```

---

## 六、一个具体示例

两个上下相邻的弦粒子：

```
A: (0,0,0)
B: (0,0,1)
```

A 的 +Z 面：

```
faceOptions[+Z] = [10, 11]
```

含义：

```
10 = 楼梯
11 = 电梯
```

B 的 -Z 面：

```
faceOptions[-Z] = [10, 11]
```

系统塌陷时：

### 情况1：选择 10

```
→ 楼梯连接上下
```

### 情况2：选择 11

```
→ 电梯连接上下
```

空间结构就此确定。

## 七、最小容器结构

```
ParticleChunk
└─ cells: list<ParticleCell>
```

Chunk 可以表示：

* 房间
* 楼层
* 建筑
* 城市区块
* 地下城区域

## 八、空间生成流程（协议级）

完整流程：

```
1. AI生成弦粒子网格
2. 每个面包含可选空间结构集合
3. 塌陷算法选择具体结构
4. 引擎根据选择结果生成真实3D模型
```

即：

```
弦粒子网络
→ 空间可选项集合
→ 塌陷选择
→ 具体三维空间
```

## 十一、从波函数到具体空间

弦粒子的完整生命周期：

```
全状态定义（波函数）
  所有可能性并存
        │
        │  塌陷
        ▼
塌陷状态（经典态）
  每面确定一个选择
        │
        │  展开
        ▼
具体三维空间
  可渲染、可交互
```

这三步对应三个独立的关注点，由不同的文档定义：

* **本文档（SPP-Core）** → 定义波函数的结构
* **[二进制协议](./03-string-particle-protocol.md)** → 定义塌陷状态的编码和放置信息
* **[引擎系统](./03-string-particle.md)** → 定义展开和渲染

存储分层、成本分析和运行效率见 [系统效率分析](./07-efficiency.md)。
