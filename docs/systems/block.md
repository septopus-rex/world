# 地块系统 (Block System)

Block（地块）是 Septopus World 的基础空间数据单元。世界场景中的所有环境和实体都是建立在离散的 Block 上的。

## 1. 基础属性与经济模型

在 Septopus 的经济与治理设计中，Block 具备明确的所有权与状态流转机制：
*   **空间尺度**：每个 Block 表示 16m × 16m 的虚拟物理空间。
*   **私有制**：每个 Block 具有唯一的所有者地址（智能合约映射）。
*   **自由交易与流转**：玩家可以对持有的地块进行自主买卖。
*   **生命周期（荒废机制）**：区块链时间驱动，若地块超过 100 链上年没有任何更新操作，该地块将进入“荒废”状态，允许任何新玩家进行占有操作。
*   **地形控制**：拥有者可以设置地块的海拔标高（Elevation），借此能够直接堆砌出高山低谷等地貌模型，同时设置预定义状态产生瀑布、岩浆等特殊效果。

## 2. Block 载体结构 (数据驱动)

在引擎内部，Block 系统通过高度抽象的加载管线流转数据。

### 数据索引
Block 的最小数据单位（Raw 数据）是用于链上极简存储的结构数组，包含四个主要位置：
```javascript
const BLOCK_RAW = [
    elevation,       // [0]: Float 标高系数
    status,          // [1]: Int   地块材质与状态
    adjuncts_array,  // [2]: Array 本 Block 上挂载的所有附属物
    game_setting     // [3]: Opt   游戏逻辑设定位
]
```

### 多态转换 (Transform Hooks)
Block 系统作为 ECS 架构的核心 Entity，定义了一系列的格式转换生命周期 Hook，用于处理它自身从“链上纯数据”到“屏幕 3D 对象”的蜕变：
*   `raw_std`：将生涩的数值索引转换成带有明确字段（如 `x, y, z, rx, material`等）的标准中间表达格式。
*   `std_3d`：将中间数据构建为能被渲染器（如 Three.js）接收渲染的 BoxGeometry，并贴上对应 status 的纹理贴图。
*   `std_border`：负责在游戏进入“编辑模式 (Edit Mode)”时，生成 Block 的边缘高亮线框（北红、南灰、东蓝、西绿以指示三维方位）。

## 3. 与附属物 (Adjunct) 的装配关系

**局部坐标化定位**：所有的附属物 (Adjunct)——例如玩家搭建的一堵墙壁、放置的一个路灯，在底层均不记录其在整个 3D 宇宙中的绝对坐标。
它们只记录它与其**所属 Block 西南角极点**的局部偏移量 `(ox, oy, oz)`。并且存储在所属 Block 的 `Adjuncts Array` 内部。 
这种“捆绑”机制极大地简化了数据的加载和销毁效率：**当玩家离开某个片区导致 Block 被卸载时，其上挂载的成百上千个小组件也会跟随 Block 数据一并被立刻回收。**

## 4. 边界预加载队列 (Queue & Loading)

世界大地图使用边缘探视预加载的方法：
1.   引擎周期性进行 `World.launch` 巡检玩家周围一定矩阵半径。
2.   遇到空洞 Block 时，推入 Block 加载队列 (BlockQueue)。
3.   加载回包后，生成实体对象并广播 `block.loaded` 核心事件，引发其他系统（如光影系统重新计算烘焙、导航网格重算等）的联动响应。
