# 事件与触发器系统 (Event & Trigger)

Septopus 的组件解耦和逻辑驱动依赖于全局统一的事件总线（Event Bus）以及实体级触发器（Trigger）。

## 1. 强类型的全局事件总线

事件系统（Event System）负责整个引擎的跨模块通讯。由于采用了 TypeScript 重构，所有的事件分类和名称被严格限定：

### 核心事件分类
| 分类 (Category) | 包含的事件 (Events) | 用途说明 |
|---|---|---|
| `system` | `init`, `launch`, `update` | 系统生命周期的流转与心跳广播 |
| `block` | `in`, `out`, `loaded`, `unload` | 与地块相关的生命周期，常用于驱动预加载管线 |
| `trigger` | `in`, `out`, `hold`, `touch` | 互动区响应（下面详述） |
| `stop` | `on`, `leave`, `beside` | 物理阻挡与站立响应 |
| `player` | `fall`, `death`, `rotate` | 本地玩家的状态变化 |
| `resource` | `parsed`, `failed` | 异步 3D 资源的加载结果 |

### 实体级精准路由 (Targeted Binding)
与传统的 DOM Event 不同，环境级事件如果进行全局广播并挨个比对目标会导致极大的性能浪费。
Septopus 事件中心支持**对象级绑定**：系统可以直接在某一个特定的实体 (如 `[2025,619]处的第0号Box`) 上去监听 `touch`。事件触发时，底层将提取对象的组合键名，仅拉起指向该实体的订阅回调。

## 2. 触发器系统 (Trigger)

触发器是游戏模式下实现世界互动的最核心也是**最危险**的组件。它本质上是一个包围盒形状的逻辑判定区。
*注意：触发器在普通浏览模式和游戏模式下工作，在 Ghost(游客/鬼魂) 模式下失效。完整的数据格式见 [触发器协议](../../protocol/cn/trigger.md)。*

### 组成结构
一个触发器由两部分构成：

- **空间体积 (Volume)**：一个不渲染网格的几何判定区（盒子 / 球体 / 圆柱），仅参与数学碰撞判定。
- **逻辑节点列表 (Logic Nodes)**：挂载在同一体积上的一组逻辑。**同一个体积可以同时绑定多个事件**（如 `in` 开门、`out` 关门），每个逻辑节点独立持有自己的事件、条件、动作与恢复动作。

### 触发器的生命周期
每个逻辑节点的执行遵循一个完整的流水线，由玩家（或者其他被监听物）的运动介入而驱动：
1.  **触发进入事件 (`trigger.in`)**：玩家坐标刚落入包围盒的一瞬间触发一次。
2.  **判断条件阶段**：只有满足前置条件（如玩家背包里拥有特定的钥匙，或自身特定的魔法值参数达标），才允许继续往下。多个条件之间为严格的 `AND` 逻辑。
3.  **执行动作阶段 (`task`)**：实际修改环境或玩家参数的行为。允许的修改方式包括：
    *   `系统环境修改`（调整天气、时间、修改 UI 提示）
    *   `附属物修改`（开门、让物体消失、升降）
    *   `玩家状态修改`（传送位置、扣除生命值等）
4.  **停留事件 (`trigger.hold`)**：当玩家在触发器内**持续停留达到逻辑节点的 `holdDuration`（毫秒）阈值后**触发，而非每帧触发。
5.  **结束退出事件 (`trigger.out`)**：玩家离开包围盒的一瞬间触发一次。
6.  **恢复动作**：当逻辑节点定义的退出条件满足后，可以选择将之前修改过的系统环境或附属物参数重置回缓存的原貌。

逻辑节点还可声明 `runOnce`，使其触发一次后即禁用。

### 多人感知的运行时
触发器体积按**参与者实体**分别记录"是否在体内"的状态。在游戏模式下接入的多个玩家（WebRTC 多人）会被独立评估，互不影响彼此的进入/停留/退出状态。

### 数据安全的权限控制 (Data Security)
触发器服从基于活动世界模式的执行权限约束：**正常模式**仅可触发环境变化与动画，**游戏模式**才可触发背包与生命值/属性修改，**Ghost 模式**全部禁用。体积亦可通过 `gameOnly` 标志声明仅在游戏模式下参与评估。

在触发器执行“修改背包元素”这类高危动作时，引擎受到强制模式约束：
由于主网区块数据防篡改，**游戏运行期间通过触发器获取及删除的物品，只在局部的虚拟内存会话中生效**，除非它由一层额外的零知识证明或合约调用被上链固化，否则单页应用的重新加载将让一切物理环境重置。
